What are CSS3 Transitions?

Circle

Transitions are one of the major additions to the latest version of CSS. Transitions are a set of animations that can be applied on HTML5 elements whenever one of their states is changed. Each element has few defined states that are linked to the interaction of the elements by the user. Live examples of most of the states are demonstrated below. CSS States


The most common states found in CSS3 are:

  • Link – This state is used to describe the appearance of a certain element whenever the property ‘href’ is used within its tag. The common use of links is with the <a> tag for linking to other webpages or parts of the document. The visited state is used in connection with the link state to define the style of a link once it has been navigated to.
  • Hover – This state is activated whenever a user hovers over the mouse pointer on the chosen element or elements. It is often used to create interactive content (roll-overs) and pull-down menus.
  • Active – This state is activated while a user is clicking on an element. It can be used to give a push button feel such as the submit button of the Virtual Assistant found on the homepage.
  • Focus – This state is frequently used for text boxes and other selectable elements. It is used to define the style of an element when it is currently selected and accepting input/interaction.

How to define a CSS State

Defining a state is very simple. It uses the same principal as defining a standard CSS declaration. The only difference is that, after the selector, the colon ‘:’ is used. This is then succeeded by the required state. The first code block in the example below defines a standard <div> element with the id property as ‘box’. The second code block defines what will happen when a user hovers over the element. The result can be shown below, hover over to see the effect!



#box { position: absolute; top: 50%; left: 50%; width: 60px; height: 60px; background: lightgreen } 

#box:hover { width: 120px; height: 120px; }

The Problem

The example on the previous slide poses a problem. As you have noticed the transition from a smaller box to a larger one is jerky and unanimated. It happens almost instantaneously. This is where CSS3 comes into place to fix problems and allow developers to design beautiful and eye-catchy websites. In order to place a smooth animation between the two CSS states the transition property is used. An image below breaks down the various part of the transition property. This is the shorthand version where multiple values are defines separated by spaces in order to reduce file-size and save time.


Add the code below to the default state of #box and hover over the element to see the animation effect. The original example has also been updated below. An image on the bottom right explains the transition property syntax.



#box { transition: all 1s 0.1s ease; }

Transition Syntax Explained:

  • Property – Which property should the transition be applied to. This is useful if you don’t want to animate certain features of an element.
  • Duration – How long should an animation run. This can be defined in seconds (s) or milliseconds (ms). The value must be a positive number.
  • Delay – This part defines how long to wait before performing the transition. It is used for more subtle effects or to prevent unnecessary transitions. This can be defined in seconds (s) or milliseconds (ms). The value must be a positive number.
  • Timing-function – This controls the speed of the transition from start to finish. The most common ones are: linear, ease, ease-in and ease-out. Linear for example has a constant speed throughout.
