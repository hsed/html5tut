CSS3 Animations

Introduction to CSS3 Animations

CSS3 animations allow up to infinite iterations of animation sequence(s) for an HTML element. They are very similar and a direct competitor to motion and shape tweens created in Adobe® Flash. CSS3 animations also use key-frames to accomplish this similar to Adobe® Flash. The only major difference is that while Adobe® use timeline controls, CSS3 animations are coded in CSS scripting language making it slight tedious to create and debug working animations. CSS3 animations are the biggest edition to CSS along with transformations.





An example of CSS3 animation can be seen on the top-left of this slide. The logo spins and scales alternatively, i.e. it first spins clockwise while zooming-in and then anticlockwise while zooming-out. An infinite option allows the animation to be played indefinitely.

Create CSS3 Animations: Define Properties [1/2]

There are two major steps involved in creating a working animation. The first step is to define the animation property and its configuration values in the code block of the element required to be animated. The next step is to define the animation in special key-frame blocks that run from 0% to 100% corresponding to the start and end of the animation. The code block on the next slide displays all of the essential and optional properties necessary to define a CSS3 animation.


Note: Important properties are commented with an ‘IMP’. You will notice the use of the ‘-webkit-’ prefix. This is required because Google® Chrome currently doesn’t support unprefixed animations. Furthermore, you will also be shown an alternative form of defining CSS3 animations which is commonly known as ‘shorthand code’.

Create CSS3 Animations: Define Properties [2/2]

The animation properties are as follows:



-webkit-animation-name: pulse;  /*IMP; name of the key-frame code block*/

-webkit-animation-duration: 2s;  /*IMP*/

-webkit-animation-timing-function: linear; /*IMP*/

-webkit-animation-delay: 1s;

-webkit-animation-iteration-count: 5; /*number of times to play the animation*/

-webkit-animation-direction: alternate;  /*which direction i.e. normal or alternate*/

-webkit-animation-fill-mode: forwards;
  
/*keeps the element in the same state as the last key-frame,

 if not specified, the element reverts back to its original design after animation end*/

-webkit-animation-play-state: running;  /*is the animation running or paused?*/


/*...............................................*/


/*alternate one line shorthand version of all of the above code just like the transition property*/

-webkit-animation: pulse 2s linear 1s 5 alternate forwards running;

Create CSS3 Animations: Define Key-frames

The next step to complete the animation definition is to define the key-frames. We use the @-webkit-keyframe selector to define a key frame and the curly brackets structure ‘{}’ just as normal CSS definitions. Inside the key-frame declaration we declare each position on the timeline. Starting from 0% and ending at 100%. Any intermediate value can be used and there is no requirement to specify all percentages but they must be in order and only used once in a single key-frame declaration. For each key-frame percentage another {} code block is created. Within this sub-block the user can define as many styles as s/he wishes to alter. It is essential that only those properties that can be animated such as transformations, opacity, colours, etc are used. An example of a pulse effect is shown below:




@-webkit-keyframes pulse {

  0% { transform: scale(1); background: #CFC; }

  50% { transform: scale(1.5); }

  100% { transform: scale(1); background: #CCF; }

  }
