Box Shadow

Box shadow is another property new in CSS3. Using box-shadow a user can add show around any CSS3 element. Box shadow is quite popular among CSS3 designers as it can be used in a variety of ways. This product also uses box-shadow in several areas. One of them is the outline between the side-menu bar and the content window. Hence box shadow can be useful for borders while giving elements a 3D elevated look. The code example is as follows:




#Box { …  box-shadow: 7px 7px 5px 0px #CCC; …}

/*Replace “…” with default properties such as size, default location, absolute position etc.*/

The syntax for the box shadow property is further explained (along with an example) on the next slide.

Box Shadow Syntax Explained

The explanation of the syntax and an example is as follows:

  • The first two values are X and Y offsets and the third one is blur as the text-shadow property.
  • The fourth value is the ‘spread’ that defines how much to expand a shadow.
  • Flexible – CSS allows developers the flexibility to change the design and layout of a webpage swiftly without changing any part of the visible content. This is really useful to create multi-themed websites.
  • Finally another value ‘inset’ can be added in the beginning and simply causes the object/box to be inside the shadow rather than the shadow being outside it which is what happens by default.

Border Radius

Border radius is a CSS3 property that allows you to create rounded shapes. Before CSS3 it was quite hard to create rounded objects without any plugins. Now this feature is supported natively in browsers and you can create anything from rounded rectangles to even semi-circles! A border radius is usually measured in either pixels or percent. The values go in clockwise order: top-left, top-right, bottom-right and then bottom-left. Border-radius is used extensively in this product e.g. the circles and rounded rectangles used for previous examples.



#Box:active { …  border-radius: 5px 5px 10px 10px; …}

/*Replace “…” with default properties such as size, default location, absolute position etc.*/

Linear Gradients

A gradient is the use of two or more colours to create a blend or pattern. This pattern can often make icons/backgrounds stand out. If used correctly they are can become an elegant part of your website. Like most other CSS3 styles, gradients need to be tested and prototyped several times before finding the perfect pattern. This can be done using HTML editors for example in the Adobe® Dreamweaver context menu, you can use the colour picker tool. There are two types of gradients linear or radial.

The angle value define in which direction should the gradient be applied e.g. top → bottom. The percentages define how much of that colour is to be used from 0% to 100%. The first colour comes on top of the second and so on. To preview the gradient effect, add the syntax in the image to a background property of an element.

Radial Gradients

Radial gradients define a gradient in a circular path. They work more like our Earth where the different layers begin and end at different radii of a defined circle. Radial gradients can also be defined in the background property of an element and can consist of two or more colour values. There is no angle requirement for a radial gradient because it is in every direction. An optional new argument is ‘shape’ which can be either circle or ellipse.
