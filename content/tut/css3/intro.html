Introduction to CSS3

What is CSS3? [1/2]

CSS (Cascade Style Sheet) is a scripting language that is used to instruct the browser on how to display a particular HTML element i.e. the appearance of a visual HTML tag. In previous tutorials, the CSS of a tag was edited using the ‘style=’ property within the first part of the tag. This is known as inline-styles. In fact, most developers around the world prefer linking style sheets to an HTML page which then contains all the styles needed for that page in one file. Style sheets is simply a user-editable file just like *.html but uses the *.css format and is written in the CSS script (explained later).

What is CSS3? [2/2]

Like HTML5, CSS3 is the newest edition of CSS that works as a subset HTML5. This means that any browser which supports HTML5 must also support CSS3 as it is part of the former. CSS3 has full backwards compatibility with earlier versions of CSS which means it has the same syntax with new selectors and properties.

The major addition to this version of CSS is animations and transitions. While the animations allow for key-frame playback using a coded structure, a transition is activated when one of the states of an HTML5 element is changed. Benefits of CSS


  • Non-redundant – The biggest benefit of CSS over traditional HTML default styling using tags is that it allows a web developer to apply the same style defined in a single file across the whole website. This solved a major issue in early years of HTML when colourful text and backgrounds were becoming popular.
  • Multiple selectors – With CSS, you can apply a particular set of styles to multiple or even all elements in one go to save the space and hassle of duplicating the same rules for different elements.
  • Flexible – CSS allows developers the flexibility to change the design and layout of a webpage swiftly without changing any part of the visible content. This is really useful to create multi-themed websites.
  • External Support – Since CSS styles are links, a webpage can load CSS stored on external servers hence allowing basic users to use pre-made templates very easily.

Syntax Explained

To define a CSS rule, the first step is to choose a selector. A browser searches for all elements that match the selector criteria and then applies the styles defined in the ‘declaration block’. The declaration block is an area enclosed by curly brackets ‘{}’.

Within a declaration block a set of properties and their corresponding values which are defined in the same way as inline-styles. A property and its value are separated by a colon ‘:’ whereas multiple pairs of properties and values are separated by a semi-colon each ‘;’. An example from W3Schools is illustrated below.
