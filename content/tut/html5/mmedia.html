The Video tag

As explained earlier it is now easier than ever before to embed a video into an HTML5 page. Unfortunately one of the major limitations is compatibility i.e. older browsers that have not native HTML5 support (for example Internet Explorer 6) will not be able to play HTML5 Video. All versions of Chrome and Firefox support <video> whereas it is only supported by Internet Explorer 9 or newer.

Multimedia

Syntax

Assuming you are familiar with the core concepts of HTML, you can use the code snippet below to add a video element to the body tag of a webpage. The syntax is very easy to follow:


  

<video id="Identifier" controls poster="" loop>

  <source  src="vid/home.mp4" type="video/mp4">

</video>

Syntax Explained [1/2]

We begin with creating a video tag. Within a video tag the actual relative or absolute path of the video the needs to be played is set. This is set using the ‘source’ tag. The source tag doesn’t require an enclosing tag as it is only one line. The path is provided in the ‘src’ property of the source tag. The type property is used to tell the browser what is the format of the video that needs to be played so that it can decode it appropriately.


Finally additional property tags can be added to the video tag for an even better experience.

Multimedia

Syntax Explained [2/2]


  • Id – This property sets a unique identifier to the video so that it can selected as an object required for CSS styling and JavaScript.
  • Controls – This property simply informs the browser to show the native controls which includes play, pause, seek etc. Although native controls vary from browser to browser, they save the hassle of additional coding and designing buttons that provide the same functionality.
  • Poster – A poster property acts like a thumbnail of the video. The poster value contains a reference of an image which is displayed at standby (before playing the video). This is quite useful in informing users what the video is about.
  • Autoplay – Generally not combined with a poster property, this simply tells a browser to automatically play the video as soon as the browser is loaded.
  • Loop – Another property which doesn’t require a value. It allows the developer to repeat a video every time it ends.

The Audio tag

The audio tag has pretty much the same syntax as the video tag except for the need of a poster. Like the video tag, all versions of Chrome and Firefox support <audio> whereas it is only supported by Internet Explorer 9 or newer. The audio tag is quite useful in creating sound effects and background scores. This product itself makes use of the audio tag for the background score and sound effects on mouse hover of menu element. A sample audio source-code can be seen below which needs to be added to a body tag in order to make it work.




<audio  id="bkgscore">

  <source src="mp3/JPong.mp3">

</audio>
